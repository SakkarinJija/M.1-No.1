# Version 7 Pilot Checklist

Complete this checklist before inviting the whole class.

## Technical Setup

- Back up the current Google Sheet and GitHub repository.
- Keep the working `config.js` with the existing `/exec` URL and class code.
- Replace `Code.gs`, restore the existing `CLASS_CODE` and `TEACHER_PIN`, run `upgradeV7`, and deploy a new Web app version.
- Hard-refresh the website and confirm the header shows Version 7.
- Test the website on one school computer and one student phone.

## Account Test

- Create two test student accounts from the Teacher Dashboard.
- Sign in as each test student using the correct access code.
- Confirm that a wrong access code is rejected.
- Reset one access code and confirm the old session/code can no longer be used after signing in again.
- Confirm that each student sees only their own learning logs, submissions, and assessments.

## Learning Workflow Test

- Save, edit, and delete a learning log.
- Create an assignment and submit evidence as a student.
- Review the submission as a teacher, add feedback, request revision, and approve the revised work.
- Verify one learning-log evidence item.
- Enter a Baseline assessment and confirm it appears on the student Skills page.

## Classroom Practice

- Explain that Journey Levels are course stages, not official CEFR certificates.
- Give access codes privately; never project or publish the class list with codes.
- Use teacher-entered assessments and reviewed evidence for progress decisions—not study time alone.
- Establish a recovery process for students who change devices or forget their code.
- Export a CSV or make a Sheet backup on a regular schedule.

## Data Protection

Do not store national identification numbers, home addresses, health information, passwords, safeguarding records, or other confidential information. Version 7 is intended for a small classroom pilot with ordinary learning records.
