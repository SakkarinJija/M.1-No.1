# Teacher Guide — Version 7

## Before Students Begin

1. Open the Teacher Dashboard.
2. Go to **Students**.
3. Create an account for each student.
4. Set a temporary access code of 6–24 characters.
5. Give each code privately to the correct student.
6. Ask students to open **Profile** and sign in.

Existing Version 6 students using the same browser may create their first code from Profile. Students changing devices need a teacher reset.

## Weekly Workflow

1. Set the announcement and weekly target in **Class Settings**.
2. Create a weekly assignment with an objective, instructions, evidence requirement, deadline, skills, and success criteria.
3. Review class activity in **Overview**.
4. Open **Assignments** and review the Submission Review Queue.
5. Mark work as **Approved**, **Needs revision**, or leave it **Submitted**. Add specific feedback and an optional score.
6. Open students who need support, verify evidence, add a support note, and record the next action.

## Assessment Workflow

Use the same rubric or equivalent tasks for Baseline and Post assessment. Enter Listening, Speaking, Reading, Writing, Vocabulary, and Grammar scores in the student detail window. Use teacher assessments—not study time alone—for formal progress decisions.

## Access Recovery

Open the student record and use **Reset Student Access**. Give the new temporary code privately. Never display a list of student access codes publicly.

## Safe Use

Use this system only for ordinary learning records. Do not enter sensitive personal, medical, safeguarding, password, or government-identification information.
